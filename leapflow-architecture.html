LeapFlow System Architecture

Signal-Driven Self-Evolving Agent Framework — Learning and Evolving from Actual Practice

Core Design Principles

  • • Signal-Driven Intelligence — learn from real-world signals
  • • Context Pipeline — Signal → Filter → Compress → Store → Retrieve → Decide
  • • Progressive Trust — STEP → CONFIRM → NOTIFY → AUTO
  • • Prediction-Error Learning — world model learns from deltas
  • • Event-Driven — typed EventBus, Protocol over ABC
  • • Dependency Inversion — core depends on abstractions

Three-Layer Architecture

  • • Intelligence Core (Python) — Engine, Copilot, World Model, Skills
  • • Platform Adaptation — MCP protocol, event normalization, capability negotiation
  • • Execution Layer — cua-driver (macOS), Mock Host, pluggable backends
  • • 7-Channel Perception — Vision, AX, FS, Clipboard, Input, App, Video
  • • DuckDB persistence — single source of truth for all state

Key Modules & Stats

  • • 18 packages · 180+ Python modules
  • • Engine: ReAct loop, graph planner, context compressor
  • • Memory: 5 providers (Working, Episodic, Semantic, Narrative, Evolution)
  • • Copilot: 4-level predictors (Hash → Markov → Embed → LLM)
  • • Security: approval workflow, risk assessment, redaction
  • • Hub: ModelScope / GitHub / HuggingFace backends
